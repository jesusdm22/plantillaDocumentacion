Agregar repositorios

Desafornutadamente, por defecto, CentOS no tiene acceso a las nuevas versiones de PHP. ¿Cómo hacemos para instalarlo?
Para ello, lo primero será agregar los repositorios EPEL y Remi, los cuales sí tienen acceso a estos paquetes.

yum install epel-release yum-utils
 yum install http://rpms.remirepo.net/enterprise/remi-release-7.rpm

Cuando tengamos los paquetes instalados, procederemos al siguiente apartado, decirle a Remi que nos de acceso a los paquetes de PHP para poder instalarlos.



Acceder al paquete de PHP 7.2

Bien, ahora, ya tenemos en nuestro CentOS los repositorios necesarios para poder realizar la descarga e instalación de la nueva versión de PHP.
Ejecutaremos:

yum-config-manager --enable remi-php72

Listo, Remi ya permite el acceso a los paquetes de PHP.



Proceso de instalación

Ahora se procede con la instalación de los paquetes. Instalaremos los paquetes más comunes necesarios para el correcto funcionamiento de PHP.
Para ello:

yum install php php-common php-opcache php-mcrypt php-cli php-gd php-curl php-mysqlnd

El servidor irá pidiendo confirmación de descarga e instalación por cada paquete, para permitir que éstos paquetes sean instalados, tendremos que confirmarlo pulsando la tecla "Y" seguida de "Enter" cada vez que nos pida confirmación.



Verificación

PHP 7.2 se encuentra ahora instalado en nuestra máquina. Para que los cambios surtan efectos, tenemos que reiniciar el servicio web Apache

systemctl restart httpd

Cuando lo reiniciemos, los cambios de PHP se habrán aplicado y la nueva versión ya estará activa en nuestro servidor, pero... ¿cómo lo comprobamos? Ejecutamos la línea:

php -v

Esta línea nos muestra la versión de PHP por pantalla. ¡Listo! PHP 7.x está ahora funcionando en su servidor.
